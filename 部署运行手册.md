## 1. 环境准备

### 1.1 安装 Python 3.14.5

**操作：**
1. 从 https://www.python.org/downloads/ 下载 Python 3.14.5 Windows 64-bit 安装包
2. 运行安装包，**勾选 "Add Python to PATH"**，点击 "Install Now"
3. 等待安装完成

**验证：**
打开命令提示符（cmd），执行：
```bash
python --version
```
**预期结果：** 输出 `Python 3.14.5`

---

**操作：**

```bash
pip install pywinauto pywin32 pynput psutil
```

## 3. 配置文件修改

### 3.1 编辑 config.jsonc

**操作：**
用记事本或 VS Code 打开 `D:\monitor\config.jsonc`，修改以下字段：

```jsonc
{
    // 替换为实际监控的群名称和群号（群号优先匹配窗口标题）
    "groups": [
        {"name": "你的群名称", "number": "你的群号"},
        {"name": "第二个群名称", "number": "第二个群号"}
    ],
    // 替换为接收通知的 QQ 联系人显示名称
    "target_contact": "采购员-李工",
    // 轮询间隔，默认 60 秒
    "poll_interval_seconds": 60,
    // 型号 CSV 文件路径
    "product_csv_path": "D:\\monitor\\products.csv",
    // 日志目录
    "log_dir": "D:\\monitor\\logs",
    // 同型号冷却时间（秒），默认 30
    "cooldown_sec": 30,
    // 是否区分大小写匹配，默认 false（不区分）
    "match_case_sensitive": false,
    // 重试次数
    "retry_attempts": 3,
    // 重试基础间隔（秒）
    "retry_delay_sec": 0.5,
    // 群间处理间隔（秒）
    "inter_group_delay_sec": 1.0,
    // 日志文件大小上限（字节，默认 10MB）
    "log_max_bytes": 10485760,
    // 日志备份保留数量
    "log_backup_count": 5
}
```

**验证：**
```bash
python -c "import json, re; raw=open('D:\\monitor\\config.jsonc','r',encoding='utf-8').read(); raw=re.sub(r'//.*','',raw); raw=re.sub(r'/\*.*?\*/','',raw,flags=re.DOTALL); cfg=json.loads(raw); print(f'配置有效：{len(cfg[\"groups\"])}个群，联系人={cfg[\"target_contact\"]}')"
```
**预期结果：** 输出类似 `配置有效：2个群，联系人=采购员-李工`，数字与你的配置一致

## 4. 产品型号 CSV 准备

### 4.1 创建 products.csv

**操作：**
在 `D:\monitor\` 下创建 `products.csv` 文件，编码推荐 UTF-8（记事本保存时选择 "UTF-8"）。格式如下：

```csv
型号,库存量
174933-1,15000
1-174936-1,1000
ABC-123,500
```

- 第一行为表头（必须包含"型号"列）
- 每行一个产品型号
- 程序自动检测编码（UTF-8-BOM / GBK 均可）

**验证：**
```bash
python -c "from matcher import ProductMatcher; m=ProductMatcher('D:\\monitor\\products.csv'); print(f'已加载 {len(m.models)} 个型号')"
```
**预期结果：** 输出 `已加载 X 个型号`（X 应与你的 CSV 行数一致）

---

### 4.2 验证型号匹配逻辑

**操作：**
```bash
python -c "
from matcher import ProductMatcher
m = ProductMatcher('D:\\monitor\\products.csv')
# 用已知型号测试匹配
result = m.match('求购174933-1现货')
print(f'匹配结果: {result}')
"
```
**预期结果：** 输出 `匹配结果: ['174933-1']`（型号转为小写）

## 5. QQ 客户端配置

### 5.1 安装 QQ NT 版

**操作：**
1. 下载 QQ NT 9.9.30+ 正式版（64位）
2. 安装到默认路径。QQ NT 版的实际安装路径取决于系统环境：
   - Windows 64位系统通常为：`C:\Program Files\Tencent\QQNT\QQ.exe`
   - 部分系统可能为：`C:\Program Files (x86)\Tencent\QQNT\QQ.exe`
3. 完成安装后启动 QQ

> **说明：** QQ 安装路径仅影响看门狗（`watchdog.py`）重启 QQ 的功能。看门狗已实现自动检测：
> 1. 优先从正在运行的 QQ.exe 进程中自动获取路径
> 2. 若 QQ 未运行，则按顺序尝试上述两个已知路径
>
> 主程序的 UI 自动化连接（`qq_automation.py`）通过窗口标题定位 QQ 进程，与安装路径无关。

**验证：**
确认 QQ.exe 文件存在：
```bash
# 检查常见路径
dir "C:\Program Files\Tencent\QQNT\QQ.exe" 2>nul || dir "C:\Program Files (x86)\Tencent\QQNT\QQ.exe" 2>nul
```
**预期结果：** 至少一个路径下存在 `QQ.exe`

---

### 5.2 登录并配置记住密码

**操作：**
1. 启动 QQ，输入账号密码
2. **勾选"记住密码"**（关键步骤，看门狗重启 QQ 后依赖此功能自动登录）
3. 点击登录，完成安全验证（如需要）
4. 登录成功后，QQ 主窗口保持可见

**验证：**
- 关闭 QQ 再重新打开，确认 QQ 能自动登录（不弹出密码输入框）
- 若重启后仍需输入密码，说明记住密码配置失败，请重新执行 5.2

---

### 5.3 打开监控群窗口

**操作：**
1. 在 QQ 主界面依次打开所有需要监控的群聊窗口
2. 每个群窗口可以**最小化到任务栏**，但**不能关闭**或**最小化到系统托盘**
3. 确认每个群窗口的标题包含配置的群号或群名称

**验证：**
```bash
python -c "from window_manager import find_window_by_title; hwnd=find_window_by_title('你的群名称'); print(f'找到窗口: {hwnd}')"
```
**预期结果：** 输出 `找到窗口: <一个数字>`（非 None），每个配置的群都应找到

---

### 5.4 确保联系人窗口可搜索

**操作：**
1. 在 QQ 主界面搜索框中输入目标联系人的名称（即 `config.jsonc` 中 `target_contact` 的值）
2. 确认搜索结果中能出现该联系人
3. 如有必要，先与该联系人进行一次对话，使聊天窗口出现在最近联系人列表中

**验证：**
- 双击搜索结果能正常打开聊天窗口
- 或者该联系人已在消息列表中可见

## 6. 系统配置

### 6.1 电源选项配置

**操作：**
1. 打开控制面板 → 硬件和声音 → 电源选项
2. 点击当前电源计划旁边的"更改计划设置"
3. "关闭显示器"设为 **"从不"**
4. "使计算机进入睡眠状态"设为 **"从不"**
5. 点击"保存修改"

**验证：**
```bash
powercfg /query | findstr /i "monitor-timeout"
```
**预期结果：** `当前电源使用方案下显示器超时设置 0x00000000 (0)` 表示从不关闭

---

### 6.2 关闭休眠

**操作：**
以管理员身份打开命令提示符，执行：
```bash
powercfg -h off
```

**验证：**
```bash
dir C:\hiberfil.sys 2>nul && echo "休眠文件仍存在" || echo "休眠已关闭"
```
**预期结果：** 输出 `休眠已关闭`

---

### 6.3 禁用锁屏

**操作（方法一：组策略，推荐）：**
1. 按 `Win+R`，输入 `gpedit.msc`，回车
2. 导航到：`计算机配置 → 管理模板 → 控制面板 → 个性化`
3. 双击"不显示锁屏"，选择 **"已启用"**，点击确定

**操作（方法二：注册表，组策略不可用时使用）：**
1. 按 `Win+R`，输入 `regedit`，回车
2. 导航到：`HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\Personalization`
3. 若 `Personalization` 键不存在，右键 `Windows` → 新建 → 项 → 命名为 `Personalization`
4. 在右侧空白处右键 → 新建 → DWORD (32位)值 → 命名为 `NoLockScreen`
5. 双击 `NoLockScreen`，将数值数据设为 `1`，点击确定

**验证：**
- 按 `Win+L` 不会进入锁屏界面（或锁屏界面被跳过直接到登录界面）

---

### 6.4 远程桌面保活配置

**操作：**
1. 按 `Win+R`，输入 `gpedit.msc`，回车
2. 导航到：`计算机配置 → 管理模板 → Windows 组件 → 远程桌面服务 → 远程桌面会话主机 → 会话时间限制`
3. 双击"设置断开会话的时间限制"，选择 **"已启用"**，时间设为 **"从不"**，点击确定
4. 双击"达到时间限制时终止会话"，选择 **"已禁用"**，点击确定
5. 关闭组策略编辑器

**验证：**
- 远程桌面连接到此服务器，关闭远程桌面窗口（不断开登录，直接关窗口）
- 等待 5 分钟后重新连接
- **预期结果：** 桌面会话保持原样，程序仍在运行，无锁屏

## 7. 演习模式验证

> **重要：此步骤在实际启动监控前必须完成。演习模式不会发送任何 QQ 消息。**

### 7.1 启动演习模式

**操作：**
```bash
cd D:\monitor
python main.py --dry-run
```

**验证：**
- 控制台无 WARNING/ERROR 输出（`WARNING` 级别以上才输出到 stderr）
- 日志文件 `logs\monitor.log` 中出现以下关键日志行：
  ```
  Connecting to QQ process (UIA backend, timeout=15s)...
  Connected to QQ process
  Loading products from ... (encoding=...)
  Loaded X product models
  初始化完成 — 监控 X 个群, X 个型号
  ```
- 日志中**不应出现** `Clipboard copy failed` 的 WARNING。程序使用 UIA 直接读取消息文本，正常情况下不再依赖剪贴板。若出现此 WARNING，说明 UIA 元素结构不匹配（QQ 版本变化），程序自动回退到剪贴板方案，详见 10.7。

> **常见问题：** 若日志中出现 `CRITICAL - QQ 连接失败`，通常是因为 `config.jsonc` 中
> `main_window_title_pattern` 配置过于宽泛（如 `.*QQ.*`）导致匹配到多个窗口。
> 默认值已设为 `^QQ$`（精确匹配 QQ 主窗口标题）。若仍有问题，检查 QQ 是否已启动并登录。

---

### 7.2 在测试群发送消息验证采集功能

**操作：**
1. 在任意监控群中发送一条包含产品型号的文本消息（如 "求购174933-1现货"）
2. 等待约 60 秒（一个轮询周期）
3. 检查日志文件

**验证：**
```bash
findstr /i "DRY-RUN" D:\monitor\logs\dry_run_verification.log
```
**预期结果：** 输出类似以下内容：
```
[DRY-RUN] 2026-05-27 14:30:12 | 群:电子元器件供需群 | 型号:174933-1 | 发送者:张三 | 原文前100字符:求购174933-1现货
```

---

### 7.3 验证去重与冷却功能

**操作：**
1. 重复发送相同型号消息
2. 等待第二、第三个轮询周期

**验证：**
```bash
findstr /c:"DRY-RUN" D:\monitor\logs\dry_run_verification.log
```
**预期结果：** 同一型号第二次出现时不会产生新的 `[DRY-RUN]` 记录（冷却期内跳过）

---

### 7.4 验证非文本消息过滤

**操作：**
1. 在监控群中发送图片、文件、贴纸
2. 等待一个轮询周期

**验证：**
- 检查日志：`findstr /i "发现.*新消息" D:\monitor\logs\monitor.log`
- **预期结果：** 图片/文件/贴纸消息不出现在新消息统计中

---

### 7.5 验证多型号合并通知

**操作：**
1. 在监控群中发送一条包含两个以上型号的消息（如 "求购174933-1和1-174936-1现货"）
2. 等待一个轮询周期

**验证：**
```bash
findstr /c:"DRY-RUN" D:\monitor\logs\dry_run_verification.log
```
**预期结果：** 一条记录，型号字段显示 `174933-1、1-174936-1`（顿号分隔合并）

---

### 7.6 验证窗口恢复功能

**操作：**
1. 将所有监控群窗口最小化到任务栏
2. 等待一个轮询周期
3. 观察群窗口是否被程序自动恢复并获取焦点

**验证：**
- 群窗口被短暂激活（可以看到窗口弹出一瞬间后焦点被切换）
- 演习日志正常记录，无 "未找到群窗口" WARNING

---

### 7.7 停止演习模式

按 `Ctrl+C` 停止程序。

**验证：**

  ```
  监控到[群名]群里出现[型号]信息：[发送者]-[时间]:
  ```

```bash
findstr /c:"[指标]" D:\monitor\logs\monitor.log

```

### 8.4 配置开机自启动

**操作（方法一：启动文件夹，推荐）：**
1. 按 `Win+R`，输入 `shell:startup`，回车
2. 在打开的文件夹中右键 → 新建 → 快捷方式
3. 输入目标：`D:\monitor\start_monitor.bat`
4. 命名快捷方式为 "QQMonitor"
5. 点击完成

**操作（方法二：任务计划程序）：**
1. 按 `Win+R`，输入 `taskschd.msc`，回车
2. 点击右侧"创建任务"
3. 常规选项卡：名称 `QQMonitor`，勾选"只在用户登录时运行"
4. 触发器 → 新建 → "登录时" → 确定
5. 操作 → 新建 → 操作"启动程序" → 程序 `pythonw.exe` → 参数 `D:\monitor\watchdog.py` → 起始于 `D:\monitor\`
6. 条件选项卡：取消勾选"只有在计算机使用交流电源时才启动此任务"

**验证：**
- 重启服务器，登录后检查进程：
  ```bash
  tasklist /fi "IMAGENAME eq pythonw.exe" /v | findstr watchdog
  ```
- **预期结果：** watchdog 进程自动启动

## 10. 故障排查

### 10.1 QQ 连接失败（CRITICAL 日志）

**现象：** 启动时日志出现 `CRITICAL - QQ 连接失败，程序退出`

**原因：** pywinauto 无法通过窗口标题连接到 QQ 进程。最常见的原因是
`ui_selectors.main_window_title_pattern` 使用了宽泛的正则表达式（如 `.*QQ.*`），
匹配到了多个包含 "QQ" 字样的窗口（VS Code 项目名、资源管理器文件夹名等），
导致 `ElementAmbiguousError`。

**排查步骤：**
1. 确认 QQ 已启动且已登录（桌面或任务栏可见 QQ 主窗口）
2. 确认 `config.jsonc` 中 `main_window_title_pattern` 为 `^QQ$`（精确匹配）。使用 `.*QQ.*` 等宽泛正则会导致 `ElementAmbiguousError`，因为它会匹配到 VS Code、资源管理器等包含"QQ"字样的其他窗口
3. 手动测试连接（程序使用三策略降级：精确标题 → 进程PID → title_re模式）：
   ```bash
   # 策略1：精确标题（推荐）
   python -c "from pywinauto import Application; app=Application(backend='uia').connect(title='QQ',timeout=10); print('连接成功')"
   # 策略2：进程PID
   python -c "import psutil; from pywinauto import Application; pids=[p.info['pid'] for p in psutil.process_iter(['name','pid']) if p.info['name']=='QQ.exe']; app=Application(backend='uia').connect(process=pids[0],timeout=10); print('连接成功')" 2>&1
   ```
4. 若步骤 3 也失败，检查 QQ 窗口是否最小化到了**系统托盘**（而非任务栏）。QQ 最小化到托盘时窗口处于离屏状态，需要先恢复窗口

**验证：** 步骤 3 的任一策略输出 `连接成功`

---

### 10.2 群窗口找不到

**现象：** 日志中出现 `未找到群窗口`

**排查步骤：**
1. 确认 QQ 中该群窗口已打开（在任务栏可见，未关闭/最小化到托盘）
2. 确认 `config.jsonc` 中群号/群名称与 QQ 窗口标题一致
3. 手动检查窗口标题：
   ```bash
   python -c "import win32gui; [print(win32gui.GetWindowText(h)) for h in [win32gui.FindWindow(None,None)] if True]"
   ```
   改为枚举所有窗口查看准确标题：
   ```bash
   python -c "import win32gui; win32gui.EnumWindows(lambda h,e: e.append(win32gui.GetWindowText(h)) if win32gui.IsWindowVisible(h) else None, (titles:=[])); [print(t) for t in titles if t]"
   ```
4. 检查群名称中是否有特殊字符导致匹配失败，尝试用群号匹配

### 10.3 QQ 自动登录失败

**现象：** `watchdog.log` 中出现 `QQ 已重启但未能自动登录`

**排查步骤：**
1. 手动打开 QQ 客户端，确认之前已勾选"记住密码"
2. 检查 QQ 是否需要验证码/设备锁验证
3. 若 QQ 版本更新，"记住密码"状态可能被重置，需重新登录勾选

### 10.4 通知未发送

**现象：** 群消息匹配成功但目标联系人未收到通知

**排查步骤：**
1. 检查日志文件是否有 `Sent notification` 记录
2. 若日志显示 `匹配成功` 但没有 `Sent notification`，检查是否被冷却跳过
3. 确认目标联系人名称（`target_contact`）与 QQ 显示名称完全一致
4. 手动在 QQ 搜索框中搜索该联系人名称，确认能找到
5. 检查 QQ 联系人搜索的 UI 选择器是否需要调整（`config.jsonc` → `ui_selectors`）

**排查步骤：**

2. 检查组策略配置是否生效：`gpupdate /force` 强制刷新组策略后重启测试

### 10.7 剪贴板复制持续失败（WARNING 日志）

**现象：** 日志中频繁出现 `WARNING - Clipboard copy failed after 3 attempts`

**原因分析：** 程序默认使用 UIA 无障碍树直接读取 QQ NT 消息文本（无需剪贴板）。若日志中出现此 WARNING，说明：

1. **UIA 消息列表元素未找到**：QQ 版本升级后 UIA 树结构变化，`Window "消息列表"` 元素不存在或名称不同。程序已自动回退到剪贴板方案（Ctrl+A/Ctrl+C）。
2. **剪贴板方案也失败**：QQ NT 基于 Electron + Direct3D 渲染，其消息区是 webview。通过自动化工具模拟的键盘快捷键（Ctrl+A/Ctrl+C）在 D3D 渲染的 Electron 窗口中无法可靠定向到消息列表——即使窗口已前台激活，按键仍然作用在输入框而非消息列表。剪贴板因此始终为空。

**诊断步骤：**
1. 检查是否出现 `UIA: '消息列表' element not found` 的 DEBUG 日志（确认 UIA 方案失败原因）：
   ```bash
   findstr /c:"消息列表" D:\monitor\logs\monitor.log
   ```
2. 手动检查 QQ 当前版本的 UIA 树结构是否变化：
   ```bash
   python -c "
   from pywinauto import Application
   from window_manager import find_window_by_title
   import win32gui, win32con, time
   hwnd = find_window_by_title('你的群名称')
   win32gui.ShowWindow(hwnd, win32con.SW_RESTORE)
   win32gui.SetForegroundWindow(hwnd)
   time.sleep(0.5)
   app = Application(backend='uia').connect(title='QQ', timeout=10)
   dlg = app.window(handle=hwnd)
   # 尝试找到消息列表
   try:
       ml = dlg.child_window(title='消息列表', control_type='Window')
       print(f'消息列表存在: {ml.exists()}')
   except:
       print('消息列表不存在，UIA 树已变化')
   # 输出所有带名称的顶层子元素
   for child in dlg.children():
       name = child.window_text()
       ctrl = child.element_info.control_type
       if name or ctrl:
           print(f'  [{ctrl}] name=\"{name}\"')
   "
   ```
3. 若 `消息列表` 仍存在但程序无法使用，检查 QQ 是否更新版本，UIA 元素的 control_type 或名称是否有变化。

**解决方案：**
- **短期**：确认 QQ 窗口未最小化到系统托盘（至少可见于任务栏）。若窗口在托盘，剪贴板方案完全无效。
- **中期**：修改 `config.jsonc` → `ui_selectors`，更新 UIA 元素定位器以匹配新版 QQ 的 UIA 树结构。
- **若确认为 QQ 版本不兼容**：回退 QQ 到已知兼容版本（QQ NT 9.9.30+），或联系开发者更新 `qq_automation.py` 中的 `_extract_messages_via_uia()` 方法以适配新版 UIA 树结构。

**操作（方法二：手动停止）：**
```bash
# 停止 main.py

# 停止 watchdog.py
taskkill /f /im pythonw.exe /fi "WINDOWTITLE eq *watchdog*"
```

**验证：**
```bash
tasklist /fi "IMAGENAME eq python.exe" | findstr main.py
tasklist /fi "IMAGENAME eq pythonw.exe" | findstr watchdog.py
```
